Comment fonctionne un QR Code ?

Un QR Code est basé sur un langage en bits. Ce dernier permet de stocker un maximum de données dans une image. Elle pourra alors permettre de transférer les données d'une personne à une autre et ainsi de diffuser des mots, numéros de téléphone, adresses, documents et autre à travers une image qui pourra être scanner par tous.
Mais comment à marche ?
Le QR Code réunit plusieurs principes :

Premier principe :
 Le QR Code a une anatomie qui s'adapte à la taille de l'image. Cette dernière est composé comme indiqué sur l'image ci-contre de plusieurs informations comme 3 patterns ainsi que des alignements en fonction de la taille et un repère "timing" qui permet la reconnaissance de la taille. Il y a aussi des données sur la version, et sur les informations formative du QR Code comme le masque utilisé ou autre information importante.
Second principe :
 Le QR Code a une fonction de codage en 3 étapes.
La première étape est une étape de codage binaire. Le codage binaire est un codage de bits correspondant à un chiffre binaire(0 ou 1) et une unité de mesure. Celui-ci correspond a un langage machine. Il est codé comme expliqué dans les images ci-contre.

Mais avant celles-ci nous devons décidé le mode de codage par exemple le mode octet (c'est-à-dire du binaire utilisant une table de codage appelés ASCII), ou le mode numérique codant seulement les nombre ou un codage spécial alpha numérique codant lettres en majuscules et nombres. Grâce à ses informations crée l'image du code binaire.

La Seconde étape est une étape de données qui consiste à calculer la longueur du mot et ainsi traduire et calculer son octet et à l'aide des informations trouvé si dessus, ajouter un module 4*4 indiquant le module de démarrage en fonction du mode choisi et un module 4*4 de finale 0,0,0,0.

La dernière étape consiste à créer grâce au code précédent, un code d'erreur qui permet d'améliorer la lisibilité du code et sa traduction lors du décodage. Ce code implique une très bonne connaissance mathématique et un très bonne connaissance informatique sur la redondance et la division de fonction polynomiale à l'aide d'indice alpha.
Troisième et Dernier Principe :
Le QR Code va pour finir être créer. Il va d'abord ajouter les données du premier principe et ensuite ajouter les données du second principe en suivant une règle de positionnement comme montrer ci-contre.

Pour finir, pour améliorer la diversité améliorer la lecture, le QR Code va être masqué, c'est-à-dire on va lui appliqués un masque qui correspondra au score de pénalités le plus faible mobilisant une meilleur redondance. Tout cela est montré dans les images ci-contre.

Pour le décodage
Lors du décodage les principes vont être inversé car il va falloir d'abord lire les données du principe 1 puis démasqué le code, et le traduire à l'envers du principe 2 et vérifier avec le code d'erreur. Pour enfin trouver la réponse et le mot de base voulu être transmis par le QR Code
